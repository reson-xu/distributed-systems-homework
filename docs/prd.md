# 产品需求文档（PRD）

## 1. 项目背景

本项目面向“分布式商品库存与秒杀系统”的课程设计场景。当前阶段以系统设计和框架说明为主，不编写具体业务代码，重点输出中文文档、最简目录结构和基础数据库表设计。

为了控制复杂度，整体采用精简版 DDD 思路：

- 不做重型 DDD
- 不强制拆成微服务
- 只保留按业务领域划分模块的思路
- 每个模块内部只保留最基础分层

## 2. 当前阶段目标

- 输出中文系统设计文档
- 明确最简目录结构
- 明确基础数据库表结构
- 输出基础 ER 图
- 输出当前阶段唯一需要设计的接口：用户登录

## 3. 当前阶段范围

系统在概念上仍包含以下四个业务模块：

- 用户模块
- 商品模块
- 库存模块
- 订单模块

但当前阶段只要求：

- 保留四个模块的目录位置
- 保留四张核心表的设计
- 只设计用户登录接口

## 4. 非目标

当前阶段不做以下内容：

- 不设计完整商品接口
- 不设计完整库存接口
- 不设计完整订单接口
- 不展开秒杀业务流程细节
- 不引入 MQ、分布式事务、分库分表等复杂方案
- 不引入重型 DDD 概念，如复杂聚合、领域事件、工厂模式

## 5. 用户角色

### 5.1 普通用户

- 登录系统

### 5.2 管理员

- 当前阶段仅作为后续扩展角色预留

## 6. 当前阶段功能需求

### 6.1 用户模块

- 提供用户登录能力

### 6.2 商品模块

- 仅保留目录和数据表，不设计接口

### 6.3 库存模块

- 仅保留目录和数据表，不设计接口

### 6.4 订单模块

- 仅保留目录和数据表，不设计接口

## 7. 当前阶段数据模型要求

数据库表命名统一采用 `t_` 前缀，当前仅保留四张核心表：

- `t_user`
- `t_product`
- `t_inventory`
- `t_order`

设计要求：

- 表结构尽量简单
- 字段以当前阶段必要内容为主
- 后续通过增量方式扩展字段或新增表

## 8. 验收标准

- 文档全部使用中文
- 架构风格体现精简版 DDD
- 目录结构清晰且足够简单
- 数据库表命名统一为 `t_*`
- API 文档中当前仅包含用户登录接口

## 9. 假设与约束

- 技术栈初步采用 Spring Boot、MyBatis、Redis、MySQL
- 当前阶段优先完成文档与框架设计
- 后续可在不推翻现有结构的前提下逐步扩展商品、库存、订单能力
